Os comandos do Giteiro

Git: Como iniciar uma pasta git

Para começar um projeto com controle de versionamento é necessário que na pasta desejada coloque o seguinte comando:

git init

Esse comando cria uma pasta oculta no seu repositório e partir dele você pode executar todos os comandos do git nesse repositório!

Config: Como configurar seu repositório

Existem vários comandos de config possíveis de executar, mas dentre os principais e muito importantes estão os seguintes: 

 git config user.name "Nome"

Esse comando define o nome que for inserido entre aspas como o nome do "Criador" desse repositório. Olha que interessante, o mesmo serve para o email conforme abaixo:

 git config user.email "Email"

Esse comando define o email que for inserido entre aspas como o email do "Criador" desse repositório.

Dica! Caso queira definir de forma que seja aplicado para todos os repositórios que iniciar em sua máquina, sem que seja necessário fazer toda vez que iniciar uma pasta git, acrescente o seguinte comando logo após a palavra config: --global

Status: Como visualizar as alterações locais

Você realizou uma série de alterações na sua pasta, e agora? Quais arquivos foram alterados? Não se lembra? Então o comando para te ajudar é o seguinte:

 git status

Toda vez que esse comando é executado ele mostra todos os arquivos que foram modificados dentro da sua pasta!

Add: Como aplicar as alterações locais

Ok, no comando anterior você aprendeu a visualizar quais arquivos sofreram alterações, muito bem. Agora é o momento de adicionar essas alterações para incluí-las de fato no que vai ser enviado para o repositório online. Você pode fazer das seguintes formas:

 git add nomeDoArquivo

Esse comando permite que você adicione um arquivo por vez, especificando o seu nome.

Agora se a intenção é agilizar o processo, esses dois comandos são utilizados para adicionar todos os arquivos que sofreram alteração na pasta de uma só vez: 

 git add .

ou

 git add *

Dica! Uma boa ideia após adicionar seus arquivos, é executar o comando git status, pois ele mostrará os arquivos que foram adicionados e os que não foram de forma separada.

Commit: Como "entiquetar" as alterações feitas

Depois de adicionar seus arquivos é hora de dar um significado à eles em conjunto. O commit serve como uma espécie de etiqueta para as alterações, nele você coloca uma mensagem significativa sobre aquela alteração que está sendo adicionada. Para isso utilizamos os comandos:

 git commit

Caso opte por esse comando, uma telinha de edição abrirá no console para que você digite a mensagem daquele commit, após terminar basta digitar :wq, que escreverá sua mensagem e sairá da tela de edição. No entanto, existe uma forma mais simples de executar esse comando, como essa abaixo: 

 git commit -m "Nome do Commit"

Nesse caso a mensagem do commit é colocada entre aspas logo após o comando -m, a vantagem é que não abre nenhuma caixa de edição! Seu commit é feito rapidinho.

Dica! Opte sempre por escrever mensagens signiticativas, e também é uma boa prática não adicionar seu projeto todo de uma vez em um único commit, mas sim fazer vários commits de acordo com alterações relevantes.

Push: Enviar alterações locais para o repositório

Bom, depois de adicionar e commitar, tá na hora de mandar as alterações pro repositório online! Para isso utlizamos o comando:

 git push linkDoRepositório

Clone: Clonar um repositório na sua máquina

Vamos supor que você está viajando e recebe uma ligação dizendo que precisa alterar um código que está incorreto no seu trabalho. Tudo bem, você tem a pasta na sua máquina desde sempre, pois foi lá que criou todo o projeto. Porém, você esqueceu de levar seu notebook na viagem! E agora? Seu colega que está com você e trouxe o dele disse que poderia te emprestar. Mas, como você acessa seu repositório na máquina dele? Muito bem, você utiliza o comando:

 git clone linkDoRepositório

Esse comando permite que você faça uma cópia do seu repositório online em qualquer outra máquina de forma local. Assim você pode realizar todas as alterações necessárias e enviá-las pro repositório online novamente! Bem legal, não é?

Pull: Atualizar a pasta local de acordo com o repositório

Lembra da viagem que você fez hipoteticamente no comando anterior? Agora você já voltou para casa e pode realizar suas mudanças no seu próprio notebook, certo? Certo, mas antes você precisa fazer uma coisinha, por que agora a sua máquina está desatualizada com o que está no repositório online! Então, para trazer toda alteração que foi feita e atualizar a pasta local da sua máquina basta utilizar o seguinte comando: 

 git pull

Branch:O que é, como utilizar

Branch é uma ramificação a partir de um repositório original. Um repositório pode ter várias branches, nelas você pode alterar como desejar os arquivos que existem sem afetar diretamente a branch original. A ideia das branches vem do pensamento de que várias pessoas podem realizar um projeto em conjunto e ao mesmo tempo, assim utilizando as branches, por exemplo, as alterações podem ser feitas de forma individual sem gerar conflitos ou sobrescrever algo que o outro fez.

Para criar uma nova branch basta digitar o seguinte comando:

 git checkout -b nomeDaBranch

Dica! Antes de executar o comando para criar uma nova branch é interessante você primeiro trocar para a branch na qual você quer que a nova branch se baseie. Para fazer isso basta executar: 

 git checkout nomeDaBranchQueQueroMeBasear

Isso é importante para que você não crie a partir de uma branch que não é a original, já que a original é sempre a mais atualizada! Logo em seguida, pode executar o comando com o -b para criar a branch nova.

O próximo passo é colocar como upstream e executar um push para a sua branch ir para o repositório remoto de fato: 

 git push --set-upstream origin nomeDaBranch

Agora se você precisar visualizar as branches locais do seu repositório, você pode listar com o comando abaixo:

 git branch

Merge: Mesclar alterações entre branches

O merge serve para mesclar as alterações que foram feitas entre as branches. Num projeto, geralmente o que é feito é sempre atualizar a sua branch particular com o que está na branch original (master), pois alguma alteração nova pode ter sido feita por alguém e você não tem ela localmente ainda. Quando o comando de merge é executado a partir do nome da branch que você quer puxar as informações, você precisa digitar uma mensagem assim como no commit. Vamos lá? Digite o comando:

 git merge nomeDaBranch

Nesse caso, uma nova janela vai abrir para que você digite a mensagem do merge. Depois é só digitar :wq que elá escreverá e fechará a caixa. E prontinho!

Ah! Para facilitar você pode digitar a mensagem diramente no comando de merge, basta utilizar esse comando:

 git merge nomeDaBranch –m  “mensagem do merge”
